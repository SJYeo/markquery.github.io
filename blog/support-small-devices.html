Support Small Devices

May 27 2013

콘텐츠에 padding과 margin 값을 적용하면 모바일에서도 똑같은 값이 유지됩니다. 작은 스크린의 미디어쿼리를 최대 @media (max-width: 767px)로 설정했을 경우 공통적으로 값을 없애야 합니다. 예를들어 span12에 padding-50의 값을 넣었을 경우, 

<div class="span12 padding-50">

767px 이하의 스크린에서는 이렇게 값을 없애버립니다.

[class*="padding"] {
  padding: 0;
}

margin 값 역시 동일하게 CSS를 설정합니다. 한 가지 단점은 있습니다. 모바일에서 텍스트가 스크린 전체를 쓰는 것보다는 여백이 20px 정도 있는 것이 콘텐츠를 읽기 좋게 해줍니다. 때문에 <p> 태그에 padding-left: 20px, padding-right: 20px의 속성을 가집니다. bootstrap의 오지리널 responsive-767px-max.less 파일을 보면,

@media (max-width: 767px) {
body {
 padding-left: 20px;
 padding-right: 20px;
}

이와 같이 left, right에 20px씩의 padding 값이 들어 있습니다. markquery에서는 이 값을 지웁니다. 왜냐하면 이미지를 스크린 가득 채울 수가 없기 때문입니다. 또한 백그라운드 컬러를 레이아웃 전체에 넣을 경우에도 모바일에서 가득 채울 수 없습니다.

// padding-left: 20px;
// padding-right: 20px;

markquery에서 사용하는 less에는 // 주석처리 되어 있습니다. 대신 p, h1 등 텍스트 관련 태그에 값을 지정해 줍니다.

또한 작은 스크린 디바이스에서 변환되는 네이게이션도 동일하게responsive-767px-max.less에 적용되야 합니다. 네비게이션은 모바일에서 드랍다운으로 변환되기 때문입니다. 

.navbar {
padding-left:20px;
padding-right:20px;
}
