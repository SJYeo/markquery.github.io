What is Markdown?

마크다운은 텍스트를 HTML로 변환하는 도구이다. HTML 대신에 읽기 쉽고 쓰기 쉬운 텍스트 문법이 특징이다. 마크다운 문법을 살펴보고, HTML 변환을 위한 도구를 소개한다.

How To Find A Great Editor

편집디자인 프로그램인 Indesign이나 QuarkXPress처럼 markquery 에디터가 있으면 좋겠지만, 에디터를 개발하는 것은 매우 오랜 시간이 필요하다. markquery 로드맵에는 에디터 개발이 포함되어 있다. 레이아웃과 디자인, 콘텐츠를 분리하고 결합할 수 있는 형태, inline 에디터와 비슷할 것이다. dreamweaver 같은 웹사이트 저작 프로그램은 절대 아니다. 하지만 콘텐츠를 위한 에디터가 당장 준비되어 있더라도 세계의 모든 CMS 플렛폼을 지원할 수 없다. 모두를 만족시킬 수 없다. 웹의 기본을 인지하자면, 결국 구조 문서인 HTML과 표현 문서인 CSS를 유저 스스로 발전시킬 수밖에 없다. 그런 의미에서 markquery 에디터는 영원히 로드맵에만 존재할지 모른다. 왜냐하면 이미 유저들을 위한 훌륭한 에디터가 있기 때문이다. HTML을 모르는 유저들을 위한 markdown 도구들과, 유용한 기능들이 많은 sublime text와 같은 에디터가 있다.

Markdown Syntax

마크다운 사용법은 너무 쉬워서 마크다운 제작자 John Gruber의 홈페이지를 방문하거나 위키피디아를 참고하는 것이 더욱 좋다. 이 문서에서는 마크다운의 편리함의 맛을 느낄 정도만 살펴볼 것이다.

  1. Daring Fireball : http://daringfireball.net/projects/markdown/

  2. wikipedia : http://en.wikipedia.org/wiki/Markdown

1. Heading

HTML에서 제목은 <h> 태그로 만들어진다. 마크다운에서는 # 하나를 텍스트 앞에 넣는 것만으로 <h1>title</h1>과 똑같은 표현이 된다.

title <h1>title</h1>

# title

HTML 제목은 중요도 순서에 따라 h1 부터 h6까지 태그를 쓸 수 있다. 마크다운에서는 # 개수로 숫자를 대신한다. <h3>subtitle</h3>는 # 세 개를 텍스트 앞에 넣으면 된다.

This is an H1 <h1>title</h1>

This is an H2 <h2>title</h2>

This is an H3 <h3>title</h3>

This is an H4 <h4>title</h4>

This is an H5 <h5>title</h5>
This is an H6 <h6>title</h6>
# This is an H1
## This is an H2
### This is an H3
#### This is an H4
##### This is an H5
###### This is an H6

2. BLOCKQUOTES

콘텐츠를 제작하다보면 인용문이 많이 사용된다. 이지윅 에디터에서 텍스트를 드레그 후 버튼을 눌렀던 것처럼 마크다운도 편리하다. >를 텍스트 앞에 넣으면 된다.

This is a blockquote with two paragraphs. Lorem ipsum dolor sit amet, consectetuer adipiscing elit. Aliquam hendrerit mi posuere lectus. Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus.

Donec sit amet nisl. Aliquam semper ipsum sit amet velit. Suspendisse id sem consectetuer libero luctus adipiscing.
<blockquote>
This is a blockquote with two paragraphs. Lorem ipsum dolor sit amet, consectetuer adipiscing elit. Aliquam hendrerit mi posuere lectus. Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus.

Donec sit amet nisl. Aliquam semper ipsum sit amet velit. Suspendisse id sem consectetuer libero luctus adipiscing.
</blockquote> 
> This is a blockquote with two paragraphs. Lorem ipsum dolor sit amet, consectetuer adipiscing elit. Aliquam hendrerit mi posuere lectus. Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus.
			  
> Donec sit amet nisl. Aliquam semper ipsum sit amet velit. Suspendisse id sem consectetuer libero luctus adipiscing.

인용문 속의 인용문(nested)는 >를 두 번 넣으면 된다.

This is the first level of quoting.
This is nested blockquote.
Back to the first level.
<blockquote>
This is the first level of quoting.
<blockquote>This is nested blockquote.</blockquote>
Back to the first level.
</blockquote> 
> This is the first level of quoting.
>> This is nested blockquote.
> Back to the first level.

3. LISTS

HTML에서 리스트 <li>는 여러 요소가 있다. 마크다운에서는 -, + 등을 텍스트 앞에 넣는다

  • Red
  • Green
  • Blue
  • Red
  • Green
  • Blue
<ul>
<li>Red</li>
<li>Green</li>
<li>Blue</li>
</ul> 
- Red
- Green
- Blue
+ Red
+ Green
+ Blud

nested lists는 tab 키를 눌러 들여쓰기를 하면 된다.

  • Red
  • Green
  • Blue
    • Red
    • Green
    • Blue
<ul>
<li>Red</li>
<li>Green</li>
<li>Blue
<ul>
<li>Red</li>
<li>Green</li>
<li>Blue</li>
</ul>
</ul> 
+   Red
+   Green
+   Blue
	-   Red
	-   Green
	-   Blue

4. Links

HTML에서 링크를 만들 떄는 <a> 태그가 사용된다. 마크다운에서는 [ ]로 링크를 만들고 () 안에 링크 urI를 넣는다.

Markquery Link
<a href="#none/">markquery link</a> 
[Markquery Link](http://markquery.com/).

5. Text Basics

텍스트에 간단한 효과를 넣는 표현 방법이다. italic으로 텍스트를 만들 때 *를 텍스트 앞뒤에 넣으면 된다. 강조를 할 경우 -를 텍스트 앞뒤에 넣는다.

this is italic and this is bold . another italic and another bold

this is important text. and percentage signs : % and %

This is a paragraph with a footnote (builtin parser only). [^note-id]

Insert [ toc ] without spaces to generate a table of contents (builtin parser only).

[^note-id]: This is the text of the note.

<p>this is <em>italic</em> and this is <strong>bold</strong> . another <em>italic</em> and another <strong>bold</strong></p>
<p>this is <code>important</code> text. and percentage signs : % and <code>%</code></p>
<p>This is a paragraph with a footnote (builtin parser only). [^note-id] </p>
<p>Insert <code>[ toc ]</code> without spaces to generate a table of contents (builtin parser only).</p>
<p>[^note-id]: This is the text of the note. </p> 
this is *italic* and this is **bold** .  another -italic- and another --bold--

this is `important` text. and percentage signs : % and `%`

This is a paragraph with a footnote (builtin parser only). [^note-id] 

Insert `[ toc ]` without spaces to generate a table of contents (builtin parser only).

[^note-id]: This is the text of the note.

6. Image

HTML에서 이미지는 <img src="/path/img.jpg" /> 태그와 같이 만들어진다. 마크다운에서는 ![이미지 설명](img path)으로 사용된다.

<img src="/docs/docs-example1.jpg" alt="example image" /> 
![example image](/docs/docs-example1.jpg)

대략 이 정도만 markdown을 사용해도 이지윅 에디터로 했던 거의 모든 것을 할 수 있다. markdown으로 콘텐츠를 만든면 문서 관리도 편리하고, 콘텐츠 수정도 훨씬 빠르게 진행될 것이다. 무엇보다 markdown은 깨끗한 HTML을 얻을 수 있다.
