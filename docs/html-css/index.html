1. Basic HTML

유저가 좋든 싫은, 웹에서 모든 콘텐츠는 HTML이다. HTML과 CSS를 전혀 모르는 유저라면, 다음 문서인 Usage로 건너뛰어 markquery를 하나씩 따라하는 것으로 시작하는 것도 좋을 것이다. HTML 파일은 head와 body로 구성된다. head에는 웹 페이지에 대한 meta 정보를 가지게 되고, body에는 콘텐츠가 쓰여진다. 자세히 알아야할 필요는 없지만 기초를 살펴보자. 문서가 HTML이라는 선언부터 시작된다. <!DOCTYPE html>은 웹 페이지가 HTML5라는 것을 뜻한다.

<!DOCTYPE html>
<head>
	<!-- place javascript and CSS in the your directory -->
	<link rel="stylesheet" href=".../style.css">
	<script src=".../script.js"></script>
</head>

<body>
	<!-- place yout contents -->
	<h1>Heading</h1>
	<p>paragraph.</p>
</body>
</html>

<head>

<head>와 </head> 사이에는 사이트 meta 정보와 CSS 파일, javascript 파일이 연결된다.

markquery.css 파일은 bootstrap.css, responsive.css 파일과 함께 하나로 압축하여 배포된다. nomargin과 pin CSS가 포함되어 있다.

<link href=".../markquery.css" rel="stylesheet">

그리고 유저의 커스텀 CSS인 style.css 파일이 링크돼야 한다. reference.css는 방대한 용량의 디자인 정보가 업데이트될 것이기 때문에 절대 웹사이트에 링크해서 사용하면 안된다. style.css는 유저가 원하는대로 파일명과 내용 수정이 가능하다. 

<link href=".../style.css" rel="stylesheet">

javascript 파일은 jquery.js 및 bootstrap.js 등 파일을 하나로 압축하여 배포한다.

<script src=".../markquery.js"></script>

<body>

요소명은 "<"와 ">" angle brackets으로 표기한다.

<h1> title </h1>
<p> contents <p>		
<strong> important! </strong>

HTML은 내용을 태그로 둘러싸는 것으로 마크업한다. 내용을 둘러싸면 내용이 어디서부터 어디까지인지 범위가 명확해진다.

<요소명>내용</요소명>

<요소명>은 시작 태그라 하고, </요소명>를 종료 태그라 한다. 종료 태그에는 /를 넣어 종료 태그임을 표기한다. 예외사항도 있는데, 콘텐츠에 이미지를 넣을 때 사용되는 <img>와 구분선을 넣는 <hr> 태그 두 개만 기억하면 된다.

<img src="/path/markquery.jpg" alt="markquery logo image" />

img 태그의 끝에 종료를 선언하는 /가 함께 쓰인다. 이처럼 <요소명 /> 형식으로 표기하는 HTML을 빈요소(empty element)라고 한다. 참고로 img 태그를 사용할 때 이미지의 경로와 함께 alt="description" 속성을 반드시 사용해야 한다. 이미지의 설명이고, 이미지를 볼 수 없을 경우에 대체 텍스트로 사용된다. 모든 img 태그에 alt 속성을 넣을 필요는 없다. 디자인에 쓰이는 이미지 혹은 내용과 관련 없는 이미지는 공백 상태로 두면 된다. alt=" "

Basic CSS

CSS 파일에 담겨있는 디자인을 부여하는 방법은 <요소명 속성명="속성값">으로 표기하는 것이다.

<h1 class="text-color-666"> title </h1>

속성명은 class이고 속성값은 text-color-666이다. markquery의 모든 속성명은 class를 사용하며, 속성값은 reference 페이지에서 얻을 수 있다. resource 페이지와 magazine 페이지에서도 소스를 얻을 수 있다.

속성값은 여러개를 사용할 수 있다. 텍스트의 컬러를 666으로, 사이즈를 20px로, line 크기를 24px로 하고 싶을 경우 하나의 class에 한 칸씩 띄어쓴다. 

<p class="text-color-666 text-size-20 text-lineheight24"> contents </p>

속성명은 중복될 수 없으나 다른 속성명을 함께 사용 가능하다. class 외에 id, href, rol 등의 속성명이 있다. 

<img src="../path/image.png" id="link" class="padding-20" alt=" " />

id는 한 페이지에 한 번만 사용 가능하다. markquery에서는 id를 거의 사용하지 않는다.

bootstrap의 기본 속성값은 - dash로 단어가 연결된다. markquery의 속성값은 - underscore로 연결한다. 

bootstrap : <div class="containter-fluid">
markquery : <div class="text-color-999">

bootstrap은 주로 레이아웃 만들 때 관련 속성값이 사용될 것이고, markquery는 편집디자인의 대부분을 차지할 것이다. underscore를 사용하는 이유는 속성값을 더블클릭 시 쉽게 복사하고 삭제할 수 있기 때문이다. dash로 연결된 단어는 하나씩만 선택된다. markquery의 대상은 모든 유저들이기 때문에 속성값의 네이밍 역시 길어질 수밖에 없다. 속성값은 네이밍 자체로 읽을 수 있는 소스로 만든다는 규칙을 가지고 있다. 단, javascript가 쓰일 경우에는 속성명을 id로 사용하고, 속성값의 단어를 dash로 연결한다.
